## Overview

The recommended way to deploy MCRIT is using the fully packaged [docker-mcrit](https://github.com/danielplohmann/docker-mcrit) solution. This approach ensures you have fully compatible versions across all components, including:

- MCRIT server and worker processes
- MongoDB database for persistence
- Web frontend for convenient interaction
- Proper networking between components

<Info>
Docker deployment is the easiest way to get started with MCRIT and is recommended for most users.
</Info>

## Prerequisites

- Docker Engine (20.10 or later)
- Docker Compose (1.29 or later)
- At least 4GB of available RAM
- Sufficient disk space for MongoDB storage

## Quick Start

<Steps>
  <Step title="Clone the docker-mcrit repository">
    ```bash
    git clone https://github.com/danielplohmann/docker-mcrit.git
    cd docker-mcrit
    ```
  </Step>

  <Step title="Start the services">
    ```bash
    docker-compose up -d
    ```
    
    This will start all required services in the background:
    - MCRIT Server (API)
    - MCRIT Worker (Job processing)
    - MongoDB (Data storage)
    - MCRIT Web (Frontend interface)
  </Step>

  <Step title="Verify the deployment">
    Access the web interface at `http://localhost:8080` or check the API status:
    
    ```bash
    curl http://localhost:8000/status
    ```
  </Step>
</Steps>

## Docker Compose Configuration

The `docker-compose.yml` file orchestrates all MCRIT components. Here's a typical configuration structure:

<Tabs>
  <Tab title="Basic Setup">
    ```yaml
    version: '3.8'
    
    services:
      mongodb:
        image: mongo:5.0
        volumes:
          - mongodb_data:/data/db
        ports:
          - "27017:27017"
        
      mcrit-server:
        image: danielplohmann/mcrit:latest
        command: mcrit server
        ports:
          - "8000:8000"
        depends_on:
          - mongodb
        environment:
          - STORAGE_SERVER=mongodb
          - STORAGE_PORT=27017
          
      mcrit-worker:
        image: danielplohmann/mcrit:latest
        command: mcrit worker
        depends_on:
          - mongodb
          - mcrit-server
        environment:
          - QUEUE_SERVER=mongodb
          - QUEUE_PORT=27017
    
    volumes:
      mongodb_data:
    ```
  </Tab>
  
  <Tab title="With Authentication">
    ```yaml
    version: '3.8'
    
    services:
      mongodb:
        image: mongo:5.0
        volumes:
          - mongodb_data:/data/db
        environment:
          - MONGO_INITDB_ROOT_USERNAME=mcrit
          - MONGO_INITDB_ROOT_PASSWORD=secure_password
        ports:
          - "27017:27017"
        
      mcrit-server:
        image: danielplohmann/mcrit:latest
        command: mcrit server
        ports:
          - "8000:8000"
        depends_on:
          - mongodb
        environment:
          - STORAGE_SERVER=mongodb
          - STORAGE_PORT=27017
          - STORAGE_MONGODB_USERNAME=mcrit
          - STORAGE_MONGODB_PASSWORD=secure_password
          - AUTH_TOKEN=your_api_token_here
          
      mcrit-worker:
        image: danielplohmann/mcrit:latest
        command: mcrit worker
        depends_on:
          - mongodb
          - mcrit-server
        environment:
          - QUEUE_SERVER=mongodb
          - QUEUE_PORT=27017
          - QUEUE_MONGODB_USERNAME=mcrit
          - QUEUE_MONGODB_PASSWORD=secure_password
    
    volumes:
      mongodb_data:
    ```
  </Tab>
</Tabs>

## Environment Variables

MCRIT components can be configured using environment variables. The most common ones are:

### Server Configuration

| Variable | Description | Default |
|----------|-------------|----------|
| `STORAGE_SERVER` | MongoDB hostname | `127.0.0.1` |
| `STORAGE_PORT` | MongoDB port | `27017` |
| `STORAGE_MONGODB_DBNAME` | Database name | `mcrit` |
| `STORAGE_MONGODB_USERNAME` | MongoDB username | `None` |
| `STORAGE_MONGODB_PASSWORD` | MongoDB password | `None` |
| `AUTH_TOKEN` | API authentication token | `""` (disabled) |

### Worker Configuration

| Variable | Description | Default |
|----------|-------------|----------|
| `QUEUE_SERVER` | MongoDB hostname | `127.0.0.1` |
| `QUEUE_PORT` | MongoDB port | `27017` |
| `QUEUE_MONGODB_DBNAME` | Database name | `mcrit` |
| `QUEUE_MONGODB_USERNAME` | MongoDB username | `None` |
| `QUEUE_MONGODB_PASSWORD` | MongoDB password | `None` |
| `QUEUE_TIMEOUT` | Job timeout in seconds | `300` |

<Note>
See the [Configuration](/deployment/configuration) page for a complete list of available configuration options.
</Note>

## Port Mappings

The default port mappings for MCRIT services are:

| Service | Internal Port | External Port | Description |
|---------|--------------|---------------|-------------|
| MCRIT Server | 8000 | 8000 | REST API endpoint |
| MongoDB | 27017 | 27017 | Database (can be hidden) |
| MCRIT Web | 3000 | 8080 | Web frontend |

<Warning>
In production deployments, consider not exposing MongoDB port 27017 externally. Only the MCRIT server and worker containers need access to it.
</Warning>

## Volume Mounts for Persistence

To ensure data persistence across container restarts, mount volumes for:

### MongoDB Data

```yaml
services:
  mongodb:
    volumes:
      - /var/lib/mcrit/mongodb:/data/db
```

### Configuration Files

You can mount custom configuration files:

```yaml
services:
  mcrit-server:
    volumes:
      - ./custom_config.py:/app/mcrit/config/McritConfig.py:ro
```

### Uploaded Files and Results

```yaml
services:
  mcrit-server:
    volumes:
      - /var/lib/mcrit/uploads:/app/uploads
      - /var/lib/mcrit/results:/app/results
```

## Managing the Deployment

### View Logs

```bash
# All services
docker-compose logs -f

# Specific service
docker-compose logs -f mcrit-server
docker-compose logs -f mcrit-worker
```

### Stop Services

```bash
docker-compose down
```

### Stop and Remove Data

```bash
docker-compose down -v
```

<Warning>
The `-v` flag will delete all volumes, including your MongoDB data. Use with caution!
</Warning>

### Update to Latest Version

```bash
docker-compose pull
docker-compose up -d
```

## Scaling Workers

For better performance with large workloads, scale the number of workers:

```bash
docker-compose up -d --scale mcrit-worker=4
```

Or define multiple workers in your `docker-compose.yml`:

```yaml
services:
  mcrit-worker-1:
    image: danielplohmann/mcrit:latest
    command: mcrit worker
    # ... configuration ...
    
  mcrit-worker-2:
    image: danielplohmann/mcrit:latest
    command: mcrit worker
    # ... configuration ...
```

## Resource Limits

Set resource limits to prevent containers from consuming all system resources:

```yaml
services:
  mcrit-worker:
    image: danielplohmann/mcrit:latest
    command: mcrit worker
    deploy:
      resources:
        limits:
          cpus: '2.0'
          memory: 4G
        reservations:
          cpus: '1.0'
          memory: 2G
```

## Health Checks

Add health checks to ensure services are running properly:

```yaml
services:
  mcrit-server:
    image: danielplohmann/mcrit:latest
    command: mcrit server
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/status"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s
```

## Troubleshooting

<Accordion title="Server not responding">
  Check if the server container is running:
  
  ```bash
  docker-compose ps
  ```
  
  View server logs:
  
  ```bash
  docker-compose logs mcrit-server
  ```
  
  Verify MongoDB connectivity from the server container:
  
  ```bash
  docker-compose exec mcrit-server ping mongodb
  ```
</Accordion>

<Accordion title="Worker not processing jobs">
  Ensure the worker can connect to MongoDB:
  
  ```bash
  docker-compose logs mcrit-worker
  ```
  
  Check for jobs in the queue by accessing the API:
  
  ```bash
  curl http://localhost:8000/jobs
  ```
</Accordion>

<Accordion title="MongoDB connection refused">
  Verify MongoDB is running:
  
  ```bash
  docker-compose ps mongodb
  ```
  
  Check MongoDB logs:
  
  ```bash
  docker-compose logs mongodb
  ```
  
  Ensure correct credentials in environment variables if authentication is enabled.
</Accordion>
